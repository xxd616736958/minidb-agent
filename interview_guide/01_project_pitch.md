# 项目介绍话术

MiniDB Agent 是我做的一个面向 PostgreSQL 数据库管理的终端智能体。它不是普通聊天机器人，而是一个可以理解数据库运维任务、规划步骤、调用 PostgreSQL 工具、执行安全检查、在写操作前请求人工审批，并最终生成交付报告和审计记录的 Agent。

这个项目的核心目标是解决数据库运维中两个常见问题：第一，DBA 或开发同学排查慢 SQL、索引、锁、表结构时需要来回查很多系统视图，流程繁琐；第二，智能体如果直接执行 SQL 风险很高，必须有明确的权限边界、审批记录和回滚/验证意识。因此我参考了 Codex 的任务循环、工具调用展示和权限恢复方式，也参考了 Claude Code 的会话协作、命令式 CLI 和上下文管理方式，把 PostgreSQL 操作封装成结构化工具，并在 Agent Loop 中加入任务理解、规划、执行、验证、错误恢复和交付报告。

如果面试官让我一句话概括，我会说：MiniDB Agent 是一个“数据库场景下的 Codex/Claude Code”，但它不是写代码，而是安全地帮人完成 PostgreSQL 诊断、调优建议、变更审批和结果交付。

我在项目里重点做了几件事：

- 把用户模糊请求转成数据库任务意图，比如健康检查、慢 SQL 诊断、表结构分析、索引建议、维护操作等。
- 设计了 PostgreSQL 工具体系，比如连接检查、表结构查看、慢 SQL 采集、EXPLAIN、索引建议、锁检查、ANALYZE、并发建索引等。
- 实现了类似 Codex/Claude 的 Agent Loop：理解任务、生成计划、调度步骤、调用工具、验证结果、失败恢复、最终交付。
- 对写操作、DDL、维护动作加了安全护栏：先只读观察，写操作必须人工审批，审批绑定 SQL hash，避免审批后执行另一条 SQL。
- 优化 CLI 使用体验：可以在目标目录直接启动，自动拉起本地服务，首次配置数据库连接，后续通过 slash command 调整配置和恢复会话。
- 做了真实 PostgreSQL 验证：连接本机数据库，发现 `public.big_orders_demo` 的慢 SQL，输出耗时、IO、临时块、建议和审计报告。

这个项目最有价值的地方不是“调用大模型”，而是把大模型放进一个可控的工程框架里：它能自己观察和决策，但每一步都被工具协议、状态机、安全策略和交付质量门约束。
